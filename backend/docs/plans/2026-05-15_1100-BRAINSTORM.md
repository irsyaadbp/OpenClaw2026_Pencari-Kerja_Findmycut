## 🎯 Core Concept

User upload foto kepala → Multi-Agent System analisis → Top 3 rekomendasi potongan rambut.

**Yang dinilai juri (30%+30%):** Bagaimana agent bekerja — autonomous reasoning, tool calls, workflow processing. Bukan UI.

## 🤖 LLM API Strategy

### 2 API Config (Cukup)

```env
# Config 1: Main LLM (Agent 2 & 3 — tool calling & reasoning)
MAIN_LLM_BASE_URL=https://xxx
MAIN_LLM_API_KEY=xxx
MAIN_LLM_MODEL=mimo-v2.5-pro

# Config 2: Vision LLM (Agent 1 — analisis foto)
VISION_LLM_BASE_URL=https://xxx
VISION_LLM_API_KEY=xxx
VISION_LLM_MODEL=xxx
```

**Kenapa cukup 2:**
- Main LLM → handle Agent 2 (Knowledge Gatherer) + Agent 3 (Ranker) — butuh tool calling support
- Vision LLM → handle Agent 1 (Vision Analyzer) — butuh image input capability
- Image generation enggak perlu — reference images pakai URL dari internet
- Semua pakai OpenAI SDK, cuma beda base URL & model

**Kenapa bukan 1 aja:**
- MiMo (main) belum tentu support vision/image input
- Kalau ternyata MiMo support vision juga, bisa disederhanakan jadi 1 config

## 🏗️ Agent Architecture

### Pipeline: Sequential (3 Agent)

```
📸 User Upload (1-4 foto)
        ↓
┌──────────────────────────────────────┐
│  🔍 Agent 1: Vision Analyzer         │
│  Input: foto URLs                    │
│  Process: panggil Vision LLM API     │
│  - analyze_image tool per foto       │
│  - merge_analyses tool               │
│  Output: FaceFeatures JSON           │
│  Tool calls: 2-5                     │
└──────────────┬───────────────────────┘
               ↓
┌──────────────────────────────────────┐
│  🎨 Agent 2: Style Matcher           │
│  Input: FaceFeatures JSON            │
│  Process: panggil Main LLM API       │
│  - query_face_shape_guide tool       │
│  - query_hair_compatibility tool     │
│  - get_style_details tool            │
│  Output: List kandidat (5-7)         │
│  Tool calls: 3-6                     │
└──────────────┬───────────────────────┘
               ↓
┌──────────────────────────────────────┐
│  ⭐ Agent 3: Ranker & Explainer      │
│  Input: FaceFeatures + kandidat list │
│  Process: panggil Main LLM API       │
│  - calculate_match_score tool        │
│  - generate_explanation tool         │
│  - get_reference_image tool          │
│  Output: Top 3 + alasan personal     │
│  Tool calls: 3-5                     │
└──────────────┬───────────────────────┘
               ↓
         📊 Result JSON
```

### Kenapa Sequential (bukan Parallel):
- Agent 2 butuh output Agent 1
- Agent 3 butuh output Agent 1 + 2
- Linear flow lebih gampang di-debug di hackathon
- Total tool calls: ~10-16 per full pipeline (impressive untuk demo)

## 🔧 Tool Definitions (Detail)

### Agent 1 Tools

```typescript
// Tool 1: Analyze single photo
analyze_image(image_url: string, angle: string)
  → Call Vision LLM API
  → Return: { face_shape, hair_thickness, hair_texture, ... }

// Tool 2: Merge multi-angle analyses
merge_analyses(analyses: FeatureAnalysis[])
  → Majority vote + average confidence
  → Return: merged FaceFeatures
```

### Agent 2 Tools

```typescript
// Tool 1: Query knowledge base by face shape
query_face_shape_guide(face_shape: string)
  → Read from hardcoded JSON
  → Return: list of recommended styles

// Tool 2: Check hair type compatibility
query_hair_compatibility(style: string, texture: string, thickness: string)
  → Read from hardcoded JSON
  → Return: { compatible: boolean, reason: string }

// Tool 3: Get full style details
get_style_details(style_name: string)
  → Read from hardcoded JSON
  → Return: { description, tips, maintenance, ... }
```

### Agent 3 Tools

```typescript
// Tool 1: Calculate match score
calculate_match_score(features: FaceFeatures, style: StyleCandidate)
  → Scoring algorithm
  → Return: { overall, face_shape, hair_type, thickness, reasons }

// Tool 2: Generate personal explanation
generate_explanation(features: FaceFeatures, style: StyleCandidate, scores: MatchScores)
  → Call Main LLM API
  → Return: { main_reason, detail_reasons, styling_tips }

// Tool 3: Find reference image URL
find_reference_image(style_name: string)
  → Search for image URL
  → Return: { image_url: string }
```

## 📁 Knowledge Base (Hardcoded JSON)

Source: barber/hairstyle knowledge. Di-hardcode karena:
- Hackathon 12 jam, enggak ada waktu scrape data
- Lebih reliable dari web search (pasti return result)
- LLM tinggal query, enggak perlu generate dari nol

### 3 File JSON:

**1. face-shapes.json**
- Mapping: face_shape → list recommended styles
- 6 bentuk wajah: oval, round, square, heart, oblong, diamond

**2. hair-types.json**
- Mapping: hair_texture + hair_thickness → compatible styles
- 4 texture: straight, wavy, curly, coily
- 3 thickness: thin, medium, thick

**3. styles.json**
- Detail per style: description, tips, maintenance, trending, reference image URL
- 12-15 styles umum

## 📡 Backend API (Bun + Hono)

### Endpoints

```
Auth:
  POST /api/auth/register
  POST /api/auth/login
  GET  /api/auth/google (optional)

Analysis:
  POST /api/analyze              → Upload foto + trigger pipeline
  GET  /api/status/:jobId        → Poll progress (agent logs)
  GET  /api/result/:jobId        → Ambil hasil akhir
```

### Flow

```
1. User login → dapat JWT
2. POST /api/analyze (Bearer JWT + multipart foto)
   → Upload foto ke R2 → dapat URLs
   → Create job di DB
   → Trigger agent pipeline (async background)
   → Return { jobId, status: "processing" }
3. FE poll GET /api/status/:jobId setiap 2 detik
   → Return agent_logs (progress real-time)
4. Pipeline selesai → status: "completed"
5. FE call GET /api/result/:jobId
   → Return FaceFeatures + Top 3 recommendations
```

## ❓ Open Questions (Perlu Diputusin)

1. **Vision LLM pakai model apa?**
   - MiMo support vision? Atau perlu model lain?
   - Kalau MiMo support, bisa 1 config aja

2. **Auth: Google OAuth atau username-password aja?**
   - Google OAuth butuh setup Google Cloud Console
   - Username-password lebih cepat buat hackathon
   - Juri enggak peduli auth method

3. **R2 atau enggak?**
   - Foto disimpan di R2 (production-style) atau
   - Foto langsung pass ke LLM tanpa upload (simpler)
   - Kalau enggak pakai R2, foto = base64 di request

4. **Rate limiting perlu?**
   - Hackathon demo, enggak ada user real
   - Tapi nunjukin awareness soal rate limiting = plus point

5. **Deploy di mana?**
   - VPS sendiri (Coolify) — udah familiar
   - Vercel/Netlify — simple tapi butuh setup
